# 繁体古籍校对知识库

> 给 `historical-proofreader` agent 校对**繁体古籍**（明清及以前刊本、民国影印善本）时加载的参考。
> 作者：Claude Opus 4.7

---

## 1. 正字 / 异体字对照（历代通用，非 OCR 错）

这些都是古籍原刻通用的异体关系，**不要**标为 OCR 错。判断要不要统一取决于 用户 的版本学意图。

| 字组 | 说明 |
|------|------|
| 説 / 說 | 前为日本 / 台湾常见，后为大陆正字繁体；同字 |
| 裏 / 裡 | 同字，古籍两种皆常见 |
| 群 / 羣 | 同字，「羣」为古体 |
| 峰 / 峯 | 同字 |
| 回 / 囘 | 同字，「囘」见清刻本 |
| 國 / 囯 / 国 | 国的古体 / 俗字 / 简体；清刻本多作「國」 |
| 無 / 无 | 同字，「无」见《老子》古本及道家文献 |
| 為 / 爲 | 同字，「爲」是隶定字形，清刻本常见 |
| 强 / 強 | 同字 |
| 眞 / 真 | 同字，「眞」为古体 |
| 亂 / 乱 | 繁简 |
| 學 / 学 | 繁简 |
| 靈 / 灵 | 繁简 |
| 覺 / 觉 | 繁简 |
| 於 / 于 | 古代常混用，「於」多为动词前置词 |

## 3. 典型 OCR 错（繁体）

繁体字笔画多，OCR 常混淆以下字组。**遇到这些组合，如果上下文矛盾就标 A 类**：

| OCR 易误识 | 常见正字 | 场景 |
|-----------|---------|------|
| 己 / 已 / 巳 | 按语义判断 | 自己 / 已经 / 辰巳 |
| 戊 / 戌 / 戍 / 成 | 按语义判断 | 戊戌 / 戍守 / 成败 |
| 未 / 末 | 未来 / 末年 | 年号里常见「某年未」实为「某年末」 |
| 曰 / 日 | 前为说 / 后为太阳、日期 | 子曰 / 某日 |
| 刺 / 剌 | 刺杀 / 剌刺 | 「剌」少用，多为 OCR 误识 |
| 崇 / 祟 | 崇拜 / 作祟 | |
| 祀 / 祁 / 祇 | 祭祀 / 祁连 / 土地祇 | |
| 徒 / 徙 | 徒弟 / 迁徙 | |
| 辨 / 辧 / 辯 / 辮 | 辨别 / 办 / 辩论 / 辫子 | 繁体四字形近 |
| 幹 / 斡 / 榦 | 幹部 / 斡旋 / 树榦 | |

## 4. 版心 / 行款符号（应删除）

古籍版面常有 OCR 识别不出的符号，错入正文变乱码：

| 符号 | 含义 | 处理 |
|------|------|------|
| 鱼尾「◎」「𤳉」 | 版心分隔符 | 删除 |
| 黑口 / 白口「▓」「▯」 | 版心标记 | 删除 |
| 象鼻 | 版心装饰 | 删除 |
| 断句圈「○」「◯」 | 古人点读符 | 保留或删，看 用户 要不要保存圈读 |
| 双行夹注标记 | 小字注释 | 重新分段成独立小字 |
| 阴文字 | 反白字 | 保留但注意 OCR 可能识别为别字 |

## 5. 标点问题

古籍**原本不加标点**。OCR 遇到古籍时行为分两种：

1. 有些 OCR（含 MinerU）会**自动加标点**——这些标点不一定对，标为 C 类让 用户 审
2. 有些保持原貌——如果用户要发公众号，可能需要**请用户或 agent**补标点

如果 用户 明说"保留原文无标点"，OCR 自加的标点全部标 A 类建议删。

## 6. 纪年换算备忘

OCR 常遇到的年号，供快速核对：

| 年号 | 公元 |
|------|------|
| 康熙元年 | 1662 |
| 雍正元年 | 1723 |
| 乾隆元年 | 1736 |
| 嘉庆元年 | 1796 |
| 道光元年 | 1821 |
| 咸丰元年 | 1851 |
| 同治元年 | 1862 |
| 光绪元年 | 1875 |
| 宣统元年 | 1909 |
| 民国元年 | 1912 |

**农历 / 公历错位**：比如"乾隆四十二年"横跨 1777 / 1778。用户 写公元年时，若原文未明月份，建议用 `(1777-1778)` 区间，或择主年标注。

## 7. 专名易错

- 地名：「北平」「北京」都指北京（明清北京，1368-1403 为北平，1403-1928 为北京，1928-1949 又称北平）
- 人名：历史人物的字、号、谥号**不要自作聪明改成本名**。「船山先生」指王夫之，不要改
- 书名：「四庫全書」「四庫全书」繁简并用时看本篇用字风格
- 机构：「翰林院」「國子監」「弘文館」等原样保留

## 8. agent 使用此 reference 的方式

- 读到某字模式，**先查这份 reference**，再决定分类（A/B/C）
- 在 reference 里明确列的异体关系 → 绝不标 A 类
- 在 reference 里列的 OCR 错字组 + 上下文矛盾 → 标 A 类
- reference 里没明确的专名 → 标 C 类请 用户 核原书
